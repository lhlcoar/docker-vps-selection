## Docker 对 VPS 有哪些硬性要求？

先别急着去比价格。Docker 对底层环境是有要求的，不满足这些，买再便宜也白搭。

**KVM 虚拟化是前提条件。** Docker 需要访问 Linux 内核的命名空间（namespace）和 cgroups 功能，只有 KVM 架构的 VPS 才能完整支持。OpenVZ 或 LXC 架构的主机对这些内核特性有限制，跑 Docker 会踩各种奇怪的坑，不推荐。

**RAM 要够用。** 一个 Docker 容器本身很轻，但你不可能只跑一个。跑 Nginx + Postgres + 一个 Node 应用，轻轻松松就用掉 1.5GB 内存。一般建议至少 2GB RAM 起步，稳稳当当跑多容器环境建议 4GB 以上。

**CPU 单核性能比核数更关键。** 很多 Docker 工作负载是单线程的——容器启动、镜像构建、脚本执行。高频 CPU 在这类场景下的优势远大于堆核数。这也是为什么业内有一批用户特别在意 CPU 主频而不是盲目追求核数。

**磁盘 I/O 不能拉胯。** 镜像拉取、日志写入、数据库读写，全靠磁盘。SSD 是基本，NVMe 更好。老式机械硬盘就别考虑了。

**Root 权限是必须的。** 没 root 就没法安装 Docker，没法管 iptables，没法搞端口映射。部分共享主机会限制这个，VPS 通常没问题，但也要确认。

## 为什么 CPU 主频对 Docker 容器特别重要？

这一点值得多说一句，因为很多人在挑 VPS for Docker 时习惯性地看核数，忽略了频率。

Docker 容器在启动时、编译镜像时、处理 HTTP 请求的单个 worker 时，都在单线程上跑。这时候给你 16 个 2.2GHz 的核，不如给你 2 个 6.0GHz 的核，实际响应速度差很多。

跑过 Geekbench 测试的人应该有感受：单核性能直接影响 web 响应延迟和任务处理吞吐量。这也是 Evoxt 在 VPS 社区里口碑不错的原因之一——他们用的 CPU 主频最高可以跑到 6.0GHz，这个数字在 VPS 市场里真的不常见。
